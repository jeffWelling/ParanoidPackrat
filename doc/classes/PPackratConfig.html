To look at the configuration

addName(configName)

addName creates a new blank configuration with the name you provided. The config is then populated by referencing the name, and using the set* functions below.

checkYourConfig()

checkYourConfig is called when an error is encountered reading the configuration.

dumpConfig()

Dump configuration

initialize()

To be called once at the beginning of the config file

length()

Number of configurations currently entered

sanityCheck(silent=nil)

This function is intended to be run AFTER the the configurations have been set to make sure the configurations entered are sane. if silent is not nil, then silent mode is activated for this function.

setBackupDestination(backup_destination)

setBackupDestination is a way of specifying a global backup Destination. Global backup destinations will be used in leu of specifying a backup destination in the individual configs by name.

backup_destination must exist and it must be a directory.

setBackupDestinationOn(name, backup_destination)

setBackupDestinationOn sets the backup destination for a specific configuration as referenced by name. There can only be one backup destination per config when specified in this method. subsequent calls simply overwrite the previous entry.

setBackupExclusions(name, exclusion)

name is the name of the config your setting.

exclusion is a file or directory that you want to exclude from the backup target. exclusion must be a subdirectory of the backup target.

setBackupTarget(name, thingToBackup)

name is the name of the config who‘s file your setting.

thingToBackup is an absolute path to the file or directory that you want backed up

setDuplication(name, guaranteed_num_of_backups)

name is the name of the config your setting.

guaranteed_num_of_backups is the guarunteed number of duplicates you want to keep across your drives. guaranteed_num_of_backups must be an integer, or a string containing an integer

Note that setting guaranteed_num_of_backups larger than the total number of backup drives your using will issue a warning, and the maximum number of duplicates possible will be stored (one per device).

setIsCritical?(name, anything_but_nil=nil)

If setIsCritical is called with a name, and anything_but_nil as anything except nil, then the backup specified by this name will be considered critical, and one copy will be kept on every drive.

setSilent()

Running PPackratConfig.setSilent will set silent mode on. Default is off.

silentMode?()

Is silent mode set to TRUE or FALSE?
