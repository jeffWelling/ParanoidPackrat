Human Browsable Dir Structure 


The backups absolutely have to be stored in a hierarchy that your average joe can navigate.  

For example, if you were to configure a backup with the name 'jeff_home', which backs up '/home/jeff' to '/mnt/sdi', the backup structure would look like;

/mnt/sdi/backup/jeff_home/2009-09-21_23:34/Documents
/mnt/sdi/backup/jeff_home/2009-09-21_23:34/Downloads
/mnt/sdi/backup/jeff_home/2009-09-21_23:34/Projects
/mnt/sdi/backup/jeff_home/2009-09-21_23:34/Projects/ParanoidPackrat
/mnt/sdi/backup/jeff_home/2009-09-21_23:34/Projects/ParanoidPackrat/...
And so on...

You can see from the example, backups are stored in backup/{backup name}/{date time of backup}/.
If you were to store multiple backups in the same backup destination, they would simply be in different {backup name} folders, because the program won't let you use two identical backup names.
